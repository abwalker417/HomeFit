# HomeFit Git Notes

Last updated: 2026-05-20 MDT

## Current product direction
- Onboarding collects only: current weight, goal weight, fitness level, days/week, limitations, equipment.
- Workout focus is NOT part of onboarding — chosen at workout-start time.
- Focus selection is radio-style (one area), not multi-select.
- Dashboard: "Start today's workout" primary CTA, "Pick one for me" secondary.
- Navigation is top-only; footer nav removed by preference.

## Current implementation status
- `workout_logic.py`
  - Equipment bug fixed: `_exercise_equipment()` now wraps strings in a list before normalizing.
  - Added `_norm()` + `_EQUIPMENT_ALIASES` to normalize underscore/dash name differences between JSON and VALID_EQUIPMENT.
  - Added `_exercise_limitations()` which reads `contraindications` field (not `limitations`) and maps to VALID_LIMITATIONS.
  - `build_workout()` now uses correct JSON fields: `default_sets`, `default_reps`, `rest_seconds`, `unit`, `id`.
  - `all_exercises_with_status()` now returns `category`, `instructions`, `available`, `reason`.
  - `pick_random_muscle_group()` now picks randomly from non-full-body muscle groups.
  - `_pick()` now shuffles pool so workouts vary each time.
- `templates/exercises.html`
  - Field names aligned with helper output (available, reason, category, instructions).
- `templates/start_workout.html`
  - Switched to radio buttons; "full body" removed from the focus list.
- `templates/workout.html`
  - Focus label shown in the eyebrow (e.g. "Day 1 · Focus: Arms").
- `static/css/style.css`
  - Removed extra `}` that was causing a CSS syntax error.

## Known open issues
1. Upper-body focus (arms/back/chest/shoulders) includes each other's exercises because all are in the "upper" category in exercises.json — no per-muscle metadata to distinguish them.
2. When user has no equipment set (empty profile), all exercises pass through the equipment filter (treated as "no preference"). This is intentional but may surprise users.
3. Onboarding/build-plan pill rows could be larger — current min-height is 48px.

## Recommended next steps
1. Add per-muscle tags to exercises.json (e.g., separate "biceps", "triceps", "chest" instead of all "full_body" / "upper").
2. Improve onboarding UI with taller stacked rows for equipment/limitations selection.
3. Add a user-facing empty-state message on the workout screen if 0 exercises are built.
