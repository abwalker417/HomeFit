# HomeFit Git Notes

Last updated: 2026-05-20 MDT

## Current product direction
- Onboarding collects only: current weight, goal weight, fitness level, days/week, limitations, equipment.
- Workout focus is NOT part of onboarding — chosen at workout-start time or via day card tap.
- Focus selection is radio-style (one area per workout), not multi-select.
- Dashboard day cards auto-build the scheduled focus (upper/lower/core/recovery) when tapped; "Change focus" pill lets user override.
- Navigation is top-only; footer nav removed by preference.
- Profile management (add/edit profiles) lives inside the Profile tab — no URL editing needed.

## Current implementation status
- `workout_logic.py`
  - Equipment bug fixed: `_exercise_equipment()` wraps strings in a list before normalizing.
  - `_norm()` + `_EQUIPMENT_ALIASES` normalize underscore/dash name differences between JSON and VALID_EQUIPMENT.
  - `_exercise_limitations()` reads `contraindications` field (not `limitations`) and maps via `_CONTRAINDICATION_TO_LIMITATION`.
  - `build_workout()` uses correct JSON fields: `default_sets`, `default_reps`, `rest_seconds`, `unit`, `id`.
  - `all_exercises_with_status()` returns `id`, `category`, `instructions`, `available`, `reason`.
  - `get_exercise_by_id()` looks up exercise by id and returns a formatted dict (used by add-exercise flow).
  - `pick_random_muscle_group()` picks from non-full-body muscle groups.
  - `_pick()` shuffles pool so workouts vary each time.
- `app.py`
  - `/build-day?label=...` maps dashboard day label to muscle groups, builds workout, saves to session.
  - `/today-workout/add` GET shows searchable exercise picker; POST appends exercise to active workout.
  - `profile_switch` sets `is_owner=True` for user_id==1 so owner can manage profiles without PIN.
  - `_LABEL_TO_MUSCLES` maps "upper body"→arms/back/chest/shoulders, "lower body"→legs/glutes, etc.
- `database.py`
  - `clear_pin(user_id)` added to support removing PINs from profile edit.
- `templates/base.html`
  - `{% block scripts %}{% endblock %}` added after app.js (was missing — silently dropped all child page scripts).
  - `defer` removed from app.js so setupLibraryFilter/startWorkout are available when block scripts execute.
- `templates/profile_edit.html`
  - Complete rewrite: PIN management (keep/set/clear radio), Manage Profiles section (owner only), Danger Zone.
- `templates/add_exercise.html`
  - New file: searchable/filterable exercise picker with `+` button; compatible with `setupLibraryFilter()`.
- `templates/dashboard.html`
  - Day cards link to `/build-day?label=...`; "Change focus" escape hatch on the workout screen.
- `static/css/style.css`
  - Removed stray `}` after media query (was causing parse error).
  - `.form label` → `.form label:not(.pill-option)` so pill-option flex-direction isn't overridden.
  - Added `.btn-danger`, `.danger-card`, profile management list styles, add-exercise styles.

## Known open issues
1. Upper-body focus includes all of arms/back/chest/shoulders — no per-muscle metadata in exercises.json to split them further.
2. No empty-state message on the workout screen when 0 exercises are generated.
3. `data/secret.key` is untracked; confirm it's in .gitignore before any public push.

## Recommended next steps
1. Add per-muscle tags to exercises.json (biceps, triceps, chest, etc. instead of coarse "upper"/"full_body").
2. Add user-facing empty-state on workout screen when 0 exercises match.
3. Consider persisting workouts to the database so users can review past sessions in Progress.
